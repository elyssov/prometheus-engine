# Задание для Аэлис: Исследование анимации движений для Prometheus Engine

## Контекст

Мы создаём собственный воксельный движок **Prometheus Engine** (Rust + wgpu). Одна из ключевых подсистем — процедурная анимация персонажей. Сейчас у нас есть:

- Воксельный солдат ОРПП на сетке 128³
- Скелетная система: 14 суставов (бёдра, колени, лодыжки, стопы, плечи, локти, кисти, голова)
- Функция `calc_leg(hip_pos, hip_angle, knee_flex, thigh_len, shin_len)` — рассчитывает позиции суставов ноги по углам
- Процедурная ходьба через `walk_at_phase(0.0-1.0)` — синусоиды на углах суставов по фазе gait cycle
- Переходы между позами через smoothstep-интерполяцию

### Текущие проблемы
1. Ходьба выглядит механически — нет естественной пружинистости
2. Колени иногда сгибаются неестественно (слишком сильно или в неправильной фазе)
3. Нет детальной кинематики стопы (перекат с пятки на носок)
4. Руки качаются слишком линейно — нет натуральной кривой
5. Присед выглядит неубедительно — нога складывается неестественно
6. Нет анимации бега, поворота, стрельбы в движении

## Что нужно исследовать

### 1. Точные углы суставов при ходьбе
Для каждой из 8 фаз gait cycle (initial contact, loading response, mid stance, terminal stance, pre-swing, initial swing, mid swing, terminal swing) нужны:
- Угол бедра (hip flexion/extension) в градусах
- Угол колена (knee flexion) в градусах
- Угол голеностопа (ankle dorsiflexion/plantarflexion) в градусах
- Угол таза (pelvic tilt, rotation, obliquity) в градусах
- Желательно — в виде таблицы или графика по фазе (0-100%)

### 2. Углы суставов при беге
То же самое но для бега. Ключевые отличия от ходьбы:
- Фаза полёта (оба ноги в воздухе)
- Больше амплитуда бедра и колена
- Больше наклон корпуса вперёд

### 3. Кинематика верхней части тела
- Как качаются руки при ходьбе/беге (углы плечевого и локтевого суставов)
- Как поворачивается торс (противовращение плеч относительно таза)
- Как двигается голова (стабилизация взгляда)

### 4. Кинематика стопы
- Перекат с пятки на носок при ходьбе (heel strike → midfoot → toe-off)
- Как это влияет на высоту лодыжки
- Угол стопы к земле в каждой фазе

### 5. Присед / стрельба с колена
- Углы суставов при приседе на одно колено (tactical kneeling)
- Как распределяется вес
- Как руки держат оружие в этой позе

### 6. Рекомендации по процедурной анимации
- Какие кривые (не линейные, а ease-in/ease-out, bezier) лучше использовать для каждого сустава
- Как реализовать IK (inverse kinematics) для ног — стопа цепляется к полу, суставы рассчитываются обратно
- Примеры open-source реализаций процедурной ходьбы (код, не только теория)

### 7. Анимация для кота (PURRGE)
Отдельно — кинематика четвероногого (кота):
- Фазы ходьбы кота (diagonal gait)
- Как двигается хвост (физическая цепочка)
- Прыжок (preparation, flight, landing)
- Красться (stealth walk — низкая стойка, медленные шаги)

## Формат результата

Идеально:
- Таблицы с углами по фазам (градусы, 0-100% цикла)
- Формулы для процедурной генерации (какие синусоиды, с какими фазовыми сдвигами)
- Ссылки на академические источники (biomechanics papers)
- Ссылки на открытые датасеты motion capture (CMU, HumanEva, etc.)
- Примеры кода (любой язык) или псевдокод

## Технические ограничения

- Наша анимация — не motion capture, а процедурная (из формул)
- Каждый кадр мы пересобираем воксельную модель из позиций суставов
- Частота анимации ~15 FPS (пересборка каждые 4 кадра при 60fps рендере)
- Модель — 128³ вокселей (будет 256³)
- Суставы задаются как (x, y, z) координаты в воксельном пространстве
- Между суставами рисуются цепочки сфер (fill_sphere вдоль вектора)
