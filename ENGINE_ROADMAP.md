## ШАГ 1: СКЕЛЕТНАЯ СИСТЕМА (`core/skeleton.rs`)

### Что это
Дерево костей. Каждая кость:
- Имя (для отладки)
- Родительская кость (или None для root)
- Локальная трансформация: позиция + вращение относительно родителя
- Длина
- Мировая трансформация (вычисляется каскадно от root)

### Ключевые операции
- `skeleton.set_bone_rotation("hip_r", pitch, yaw, roll)` — задать вращение
- `skeleton.solve_forward()` — пересчитать все мировые позиции от root вниз
- `skeleton.get_world_pos("wrist_r")` — получить мировую позицию сустава
- `skeleton.get_world_transform("hand_r")` — позиция + ориентация (для привязки оружия)

### Скелет человека (14+ костей)
```
Root (pelvis)
├── spine
│   ├── chest
│   │   ├── neck
│   │   │   └── head
│   │   ├── shoulder_l
│   │   │   ├── upper_arm_l
│   │   │   │   └── forearm_l
│   │   │   │       └── hand_l
│   │   └── shoulder_r
│   │       ├── upper_arm_r
│   │       │   └── forearm_r
│   │       │       └── hand_r
│   ├── hip_l
│   │   ├── thigh_l
│   │   │   └── shin_l
│   │   │       └── foot_l
│   └── hip_r
│       ├── thigh_r
│       │   └── shin_r
│       │       └── foot_r
```

### Важно
Вращения — кватернионы (не эйлеровы углы). Эйлеровы дают gimbal lock. Кватернионы дают плавную интерполяцию (slerp).

## ШАГ 2: ПОЗА (`core/pose.rs`)

### Что это
Набор вращений для каждой кости скелета в один момент времени.

```rust
struct Pose {
    rotations: HashMap<BoneId, Quat>,  // кватернион для каждой кости
    root_position: Vec3,                // позиция корня в мире
    root_rotation: Quat,                // вращение всего тела (поворот к цели)
}
```

### Ключевые операции
- `Pose::standing()` — стоит прямо
- `Pose::shooting()` — руки подняты к плечу
- `Pose::kneeling()` — на колене
- `Pose::lerp(a, b, t)` — интерполяция между позами (slerp для каждой кости)
- `Pose::apply(skeleton)` — применить позу к скелету

## ШАГ 3: ВОКСЕЛЬНОЕ ТЕЛО (`core/body.rs`)

### Что это
Берёт скелет (с мировыми позициями), заполняет воксели вокруг костей.

### Алгоритм
Для каждой кости:
1. Получить мировые позиции начала и конца
2. Получить радиус (задаётся при создании: "thigh_r" = radius 10)
3. Заполнить сферами вдоль линии (уже работает — `fill_limb`)
4. Каждая кость имеет material/color

### Важно
Body НЕ знает о позе или анимации. Он просто берёт текущие мировые позиции из скелета и рисует. Разделение ответственности.

## ШАГ 4: IK — INVERSE KINEMATICS (`core/ik.rs`)

### Two-Bone IK (для ног)
Входные данные: позиция бедра + целевая позиция стопы.
Выходные: углы бедра и колена.
Закон косинусов. Колено всегда указывает вперёд (pole vector).

### Aim IK (для стрельбы)
Входные данные: позиция плеча + целевая точка (враг/мебель).
Выходные: вращения spine → chest → shoulder → upper_arm → forearm.
Цепочка костей поворачивается чтобы hand_r (и привязанное оружие) смотрели на цель.

### Foot IK (стопа на земле)
Стопа привязана к позиции на полу. Тело движется — ноги адаптируются через IK.

## ШАГ 5: ПРИВЯЗКИ (`core/attachment.rs`)

### Что это
Объекты, привязанные к костям: оружие, снаряжение, шлем.

```rust
struct Attachment {
    parent_bone: BoneId,          // к какой кости привязан
    local_offset: Vec3,           // смещение от кости
    local_rotation: Quat,         // вращение относительно кости
    voxel_model: VoxelModel,      // воксельная модель объекта
    attach_points: Vec<AttachPoint>, // точки на объекте (дуло, прицел)
}

struct AttachPoint {
    name: String,        // "muzzle", "scope", "grip"
    local_pos: Vec3,     // позиция на модели
}
```

### Как работает
1. Оружие привязано к `hand_r` с offset и rotation
2. При каждом обновлении: мировая позиция оружия = мировая позиция hand_r × local_transform
3. `attachment.get_world_point("muzzle")` → мировая позиция дула
4. Пуля создаётся в позиции дула, летит в направлении ствола

### Ключевое
Солдат поворачивается → рука поворачивается → оружие поворачивается → дуло смотрит на цель → пуля летит из дула. ВСЁ АВТОМАТИЧЕСКИ. Я не считаю позицию пули. Движок считает.

## ШАГ 6: ENTITY (`core/entity.rs`)

### Что это
Игровой объект = скелет + тело + привязки + состояние.

```rust
struct Entity {
    skeleton: Skeleton,
    body_def: BodyDefinition,     // радиусы костей, материалы
    attachments: Vec<Attachment>,
    position: Vec3,               // позиция в мире
    rotation: Quat,               // вращение в мире (поворот к цели)
    current_pose: Pose,
    target_pose: Pose,
    blend_speed: f32,
}
```

### Ключевые операции
- `entity.look_at(target)` — повернуть всё тело к цели
- `entity.aim_at(target)` — нацелить оружие на точку (через aim IK)
- `entity.set_pose(Pose::shooting())` — перейти в позу (с блендингом)
- `entity.fire()` — создать пулю из attachment point "muzzle"
- `entity.update(dt)` — обновить блендинг, IK, мировые позиции
- `entity.rasterize(grid)` — отрисовать в воксельный grid

## ШАГ 7: ФИЗИКА ПУЛЬ (`physics/bullet.rs`)

### Что это
Пуля = позиция + направление + скорость. Каждый кадр:
1. Двигается вперёд
2. Raycast по grid: есть ли воксель на пути?
3. Если да → destruction.rs

### Визуализация
Трассер = цепочка ярких вокселей за пулей (длина 5-10 вокселей).
Дульная вспышка = яркие воксели у muzzle на 2-3 кадра.

---

## ШАГ 8: РАЗРУШЕНИЕ (`physics/destruction.rs`)

### Алгоритм
1. Точка попадания → удалить воксели в сфере (радиус зависит от калибра)
2. Проверить связность: flood fill от центра масс объекта
3. Отделившиеся куски → новые объекты (обломки)
4. Обломки получают физику: скорость от импульса, гравитация
5. Мелкие обломки (<N вокселей) → частицы

### Для v0.1
Упрощённо: только удаление вокселей + частицы. Без connectivity check и отделения обломков. Это на v0.2.

## ШАГ 9: СЦЕНА (`scene/world.rs`)

### Что это
Мир = один большой Grid + список Entity.

```rust
struct World {
    grid: Grid,                    // статичная геометрия (стены, пол, мебель)
    entities: Vec<Entity>,         // динамические объекты (солдат, кот)
    bullets: Vec<Bullet>,
}
```

### Обновление каждый кадр
1. Обновить Entity (анимация, IK, позы)
2. Обновить пули (движение, коллизии)
3. Очистить grid от предыдущих Entity-вокселей
4. Растеризовать Entity в grid
5. Upload grid на GPU
6. Render
