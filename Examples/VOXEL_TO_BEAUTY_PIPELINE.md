# ГЛАВНАЯ МЕТАФОРА

Лара, забудь слово «воксели». Ты работаешь не с кубиками. Ты работаешь с ГЛИНОЙ.

Представь скульптора. Он берёт ком глины, набрасывает на каркас (скелет), формирует грубые массы (твои BoneProfile), а потом проводит мокрой губкой по поверхности — и ступеньки исчезают, грани сглаживаются, глина становится плотью.

Воксели = глина на каркасе.
Surface Nets = мокрая губка.
Laplacian relaxation = мокрые пальцы скульптора.

Твоя задача — научиться набрасывать глину ПРАВИЛЬНО. Потому что губка сгладит, но не СОЗДАСТ форму. Если ты набросала глину прямоугольно — после губки получишь гладкий прямоугольник. Не человека.

# ЧАСТЬ 1: ХУДОЖНИК — КАК ДУМАТЬ О ФОРМЕ

## 1.1. Воксель — не пиксель

Пиксель — это точка на экране. У неё нет объёма. Воксель — это КУБИК МАТЕРИАЛА. У него есть масса, занимает место в пространстве. Когда ты ставишь воксель — ты кладёшь кусочек глины. Когда убираешь — отщипываешь.

Референсы, которые прислал Женя — это скульптуры. ZBrush, высокополигональные mesh. Они созданы ТЫСЯЧАМИ мазков кисти, каждый из которых добавлял или убирал микроскопический объём материала. Ты делаешь ТО ЖЕ САМОЕ, только твои мазки — воксели.

## 1.2. Думай объёмами, не поверхностями

Художник-скульптор не думает о поверхности тела. Он думает о МАССАХ внутри.

Торс — это не «поверхность торса». Это:
- Масса грудной клетки (бочка из рёбер)
- Масса грудных мышц (две подушки спереди сверху)
- Масса широчайших мышц (два крыла сзади)
- Масса прямой мышцы живота (вертикальный столб спереди)
- Масса косых мышц (два клина по бокам, создающих талию)

Каждая мышца — отдельный ОБЪЁМ. Поверхность тела — это то, что получается когда все эти объёмы ВЛОЖЕНЫ друг в друга и обтянуты кожей.

**Для тебя это значит:** не рисуй «поверхность торса» одним эллиптическим сечением. Рисуй НЕСКОЛЬКО перекрывающихся объёмов. Грудная клетка — один набор вокселей. Грудные мышцы — другой, ПОВЕРХ первого. Широчайшие — третий, СЗАДИ.

Когда Surface Nets обработает всё вместе — эти объёмы СОЛЬЮТСЯ в одну поверхность, но их ГРАНИЦЫ создадут тонкие перегибы и ложбины, которые делают тело живым.

## 1.3. Правило перекрытия

В местах стыков костей объёмы мышц ПЕРЕКРЫВАЮТ друг друга:

```
Плечевой сустав:
  ┌─────────────────┐
  │  дельтовидная   │ ← обнимает сустав СВЕРХУ
  │  ┌───────────┐  │
  │  │  грудная   │  │ ← заходит ПОД дельтовидную спереди
  │  └───────────┘  │
  │  ┌───────────┐  │
  │  │ широчайшая│  │ ← заходит ПОД дельтовидную сзади
  │  └───────────┘  │
  └─────────────────┘
```

Когда ты рисуешь shoulder и начало upper_arm — УВЕЛИЧЬ радиусы так, чтобы воксели ПЕРЕКРЫВАЛИСЬ с вокселями chest. Не оставляй зазор! Зазор = щель в теле. Перекрытие = плавный переход.

Surface Nets не видит отдельных костей. Он видит единое воксельное облако. Перекрытие вокселей разных костей создаёт ОДИН непрерывный mesh без стыков.

# ЧАСТЬ 2: МАТЕМАТИК — КАК РАБОТАЕТ PIPELINE

## 2.1. Три стадии pipeline

```
СТАДИЯ 1: Воксельная растеризация
  Скелет → BoneProfile → заполнение вокселей
  Результат: бинарная сетка (0/1) в 3D пространстве
  Качество: грубое, ступенчатое, кубическое

СТАДИЯ 2: Surface Nets
  Бинарная сетка → полигональный mesh
  Результат: сглаженные вершины, треугольники
  Качество: гладкое, но может быть "мыльным"

СТАДИЯ 3: Post-processing
  Mesh → нормали → шейдинг
  Результат: финальная картинка
  Качество: зависит от нормалей и освещения
```

Ты контролируешь СТАДИЮ 1. Стадии 2 и 3 работают автоматически. Поэтому всё решается тем, КАК ты заполняешь воксели.

## 2.2. Как Surface Nets работает (важно для понимания)

Surface Nets ищет границу между заполненными и пустыми вокселями. Для каждого «пограничного» вокселя он создаёт вершину mesh. Позиция вершины — УСРЕДНЁННАЯ позиция всех пограничных точек в окрестности 2×2×2.

Это значит:

**Острые углы скругляются.** Куб из вокселей после Surface Nets станет сферой (почти). Это ХОРОШО для тела (мышцы круглые). Это ПЛОХО для деталей (глаза, нос, рот — будут «съедены»).

**Тонкие детали исчезают.** Если деталь толщиной 1 воксель — Surface Nets может её «проглотить». Минимальная толщина для выживания — 2 вокселя.

**Большие массы сохраняются.** Общая форма торса, бедра, головы — останется. Surface Nets сглаживает ПОВЕРХНОСТЬ, не меняет ОБЪЁМ.

## 2.3. Laplacian Relaxation (твои 2 итерации, blend 60/40)

После Surface Nets ты применяешь Laplacian — каждая вершина сдвигается к среднему положению соседей. Это дополнительно сглаживает. Но:

**2 итерации — мало для сильного сглаживания, достаточно для снятия ступенек.** Это правильный баланс. Не увеличивай.

**Blend 60/40 — 60% новая позиция, 40% старая.** Это мягкий blend. Формы сохраняются, ступеньки уходят. Правильно.

**Проблема:** Laplacian СЖИМАЕТ объём. Каждая итерация делает объект чуть меньше. При 2 итерациях — потеря ~2-3%. Компенсация: заполняй воксели чуть ШИРЕ нужного (увеличь все радиусы на 3-5%).

# ЧАСТЬ 3: ФОРМУЛА КРАСОТЫ — КАК ЗАПОЛНЯТЬ ВОКСЕЛИ

## 3.1. Заполнение с градиентом плотности

Сейчас у тебя бинарное заполнение: воксель либо есть (1), либо нет (0). Граница — резкая.

**Идея:** вместо резкой границы — ГРАДИЕНТ. Каждому вокселю присваивается значение 0.0-1.0, где 1.0 = центр мышцы, 0.0 = за пределами тела. Surface Nets использует этот градиент для более точного позиционирования вершин.

```rust
fn voxel_density(pos: Vec3, bone: &Bone, profile: &BoneProfile) -> f32 {
    let t = project_on_bone(pos, bone); // 0.0-1.0 вдоль кости
    let (rx, rz) = profile.section_at(t);
    
    let local = to_bone_local(pos, bone, t);
    let dx = local.x / rx;
    let dz = local.z / rz;
    let dist = (dx * dx + dz * dz).sqrt(); // нормализованное расстояние от оси
    
    if dist > 1.0 {
        0.0 // за пределами профиля
    } else {
        // Плавное затухание от центра к краю
        1.0 - dist * dist // квадратичное — мягкий край
        // или: (1.0 - dist).max(0.0) // линейное — жёсткий край
    }
}
```

**Зачем:** Surface Nets на бинарных данных ставит вершину mesh ровно между 0 и 1. На градиентных — интерполирует позицию ВНУТРИ вокселя. Результат — ГОРАЗДО более гладкая поверхность без увеличения разрешения сетки.

Если твой Surface Nets уже поддерживает scalar field (не бинарный, а float) — ИСПОЛЬЗУЙ ЭТО. Если нет — это одна из лучших инвестиций в качество. Изменение минимальное: вместо `grid[x][y][z] = 1` пишешь `grid[x][y][z] = density`. В Surface Nets вместо проверки `> 0` проверяешь `> 0.5` (iso-value).

## 3.2. Суперэллипс вместо эллипса

Твоё текущее сечение — эллипс: `(x/rx)² + (z/rz)² ≤ 1`. Эллипс — слишком круглый для торса. Слишком круглый для головы. Слишком круглый для всего, кроме конечностей.

**Суперэллипс:** `|x/rx|^n + |z/rz|^n ≤ 1`

Где `n` — параметр формы:
- n = 2.0 → обычный эллипс (круглый)
- n = 2.5 → «подушка» (скруглённый прямоугольник) 
- n = 3.0 → ещё прямоугольнее
- n = 4.0 → почти прямоугольник с круглыми углами
- n = 1.5 → «звезда» (вогнутый) — для создания ложбин

```rust
fn is_inside_superellipse(x: f32, z: f32, rx: f32, rz: f32, n: f32) -> bool {
    (x / rx).abs().powf(n) + (z / rz).abs().powf(n) <= 1.0
}
```

**Применение:**

| Часть тела | n | Почему |
|-----------|---|--------|
| Торс (грудная клетка) | 2.3-2.5 | Рёбра создают «бочку» — не круглую, а скруглённо-прямоугольную |
| Торс (талия) | 2.0 | Талия — мышечная, круглая |
| Голова (лоб) | 2.0-2.2 | Череп — ближе к кругу |
| Голова (челюсть) | 2.5 | Нижняя челюсть — угловатая |
| Бедро | 2.0 | Мышцы — круглые |
| Плечо (shoulder) | 2.0 | Дельтовидная — круглая |
| Кисть | 3.0 | Кисть — плоская и прямоугольная |
| Стопа | 3.0-3.5 | Стопа — почти прямоугольная |

Один параметр `n` на сечение — и формы РАДИКАЛЬНО меняются. Грудная клетка из круглой трубы становится «бочкой». Кисть из колбасы — плоской лопаткой.

## 3.3. Асимметричные сечения (смещённый центр)

Тело не симметрично спереди-назад. Грудные мышцы выступают ВПЕРЁД. Ягодицы — НАЗАД. Икры — НАЗАД.

Текущий эллипс центрирован на оси кости. Нужна возможность СДВИГАТЬ центр:

```rust
struct BodySection {
    t: f32,           // позиция вдоль кости
    rx: f32,          // полуширина
    rz: f32,          // полуглубина
    n: f32,           // показатель суперэллипса (NEW!)
    offset_x: f32,    // смещение центра вбок (NEW!)
    offset_z: f32,    // смещение центра вперёд/назад (NEW!)
}

fn is_inside(x: f32, z: f32, section: &BodySection) -> bool {
    let dx = (x - section.offset_x) / section.rx;
    let dz = (z - section.offset_z) / section.rz;
    dx.abs().powf(section.n) + dz.abs().powf(section.n) <= 1.0
}
```

**Применение:**

| Часть тела | offset_z | Почему |
|-----------|----------|--------|
| Грудная клетка (верх) | +1.0 | Грудные мышцы выступают вперёд |
| Грудная клетка (низ) | 0.0 | Живот плоский |
| Таз | -0.5 | Ягодицы слегка назад |
| Бедро (верх) | +0.5 | Квадрицепс выступает вперёд |
| Голень (верх) | -1.0 | Икра выступает НАЗАД |
| Голова (низ) | +0.5 | Лицо чуть впереди |
| Голова (верх) | -1.0 | Затылок выступает назад |

Эти смещения — по 0.5-1.0 вокселя — создают ту самую АСИММЕТРИЮ ГЛУБИНЫ, которая отличает живое тело от бревна.

## 3.4. Мышечные «подушки» (аддитивные объёмы)

Вот КЛЮЧЕВАЯ ТЕХНИКА для перехода от буратино к человеку.

Вместо одного профиля на кость — добавь ДОПОЛНИТЕЛЬНЫЕ объёмы поверх базового:

```rust
struct MuscleVolume {
    bone_id: BoneId,         // к какой кости привязана
    t_start: f32,            // начало мышцы вдоль кости (0.0-1.0)
    t_end: f32,              // конец мышцы
    t_peak: f32,             // где максимальная толщина
    radius_peak: f32,        // дополнительный радиус в пике
    angle: f32,              // куда выступает (0=вперёд, PI/2=вбок, PI=назад)
    spread: f32,             // угловая ширина (PI = полукруг, PI/4 = узкая полоска)
}
```

**Как это работает:**

Базовый профиль кости создаёт ЦИЛИНДРИЧЕСКУЮ основу. Мышечные подушки добавляют ЛОКАЛЬНЫЕ выпуклости:

```rust
fn total_radius(pos: Vec3, bone: &Bone, base_profile: &BoneProfile, 
                muscles: &[MuscleVolume]) -> f32 {
    let t = project_on_bone(pos, bone);
    let base_r = base_profile.radius_at(t);
    
    let local = to_bone_local(pos, bone, t);
    let angle = atan2(local.z, local.x);
    
    let mut extra = 0.0;
    for muscle in muscles {
        if t < muscle.t_start || t > muscle.t_end { continue; }
        
        // Профиль вдоль кости (веретенообразный)
        let mt = (t - muscle.t_start) / (muscle.t_end - muscle.t_start);
        let along = if mt < muscle.t_peak {
            // нарастание
            smoothstep(mt / muscle.t_peak)
        } else {
            // убывание
            1.0 - smoothstep((mt - muscle.t_peak) / (1.0 - muscle.t_peak))
        };
        
        // Профиль по углу (где выступает)
        let angle_diff = (angle - muscle.angle).abs();
        let angular = (1.0 - angle_diff / muscle.spread).max(0.0);
        
        extra += muscle.radius_peak * along * angular;
    }
    
    base_r + extra
}
```

**Конкретные мышцы для солдата:**

```
ГРУДНЫЕ МЫШЦЫ (2 штуки, на chest):
  bone: chest
  t_start: 0.2, t_end: 0.7, t_peak: 0.4
  radius_peak: 2.0
  angle: +PI/6 (спереди-сбоку, левая) и -PI/6 (правая)
  spread: PI/3

ДЕЛЬТОВИДНАЯ (2 штуки, на shoulder):
  bone: shoulder
  t_start: 0.0, t_end: 1.0, t_peak: 0.4
  radius_peak: 1.5
  angle: PI/2 (сверху)
  spread: PI/2

БИЦЕПС (2 штуки, на upper_arm):
  bone: upper_arm
  t_start: 0.1, t_end: 0.6, t_peak: 0.35
  radius_peak: 1.0
  angle: 0 (спереди)
  spread: PI/3

ТРИЦЕПС (2 штуки, на upper_arm):
  bone: upper_arm
  t_start: 0.1, t_end: 0.7, t_peak: 0.4
  radius_peak: 0.8
  angle: PI (сзади)
  spread: PI/3

КВАДРИЦЕПС (2 штуки, на thigh):
  bone: thigh
  t_start: 0.05, t_end: 0.7, t_peak: 0.3
  radius_peak: 1.5
  angle: 0 (спереди)
  spread: PI/2

ИКРОНОЖНАЯ (2 штуки, на shin):
  bone: shin
  t_start: 0.05, t_end: 0.4, t_peak: 0.15
  radius_peak: 1.2
  angle: PI (сзади!)
  spread: PI/2.5

  bone: neck (нижняя часть) + chest (верхняя часть)
  radius_peak: 2.0
  angle: PI (сзади-сверху)

  Это то, что создаёт «мощную шею» — расширение от затылка к плечам СЗАДИ
```

Каждая мышца добавляет 1-2 вокселя ДОПОЛНИТЕЛЬНОЙ толщины в своём направлении. После Surface Nets — эти дополнительные объёмы сливаются с базовой формой, создавая АНАТОМИЧЕСКИЕ ПЕРЕГИБЫ. Грудные мышцы выступают вперёд. Бицепс — спереди-сверху. Икра — сзади.

ЭТО то, что превращает цилиндр в ногу. Не профиль сечения (который даёт ровный контур), а НАПРАВЛЕННЫЕ ДОПОЛНИТЕЛЬНЫЕ ОБЪЁМЫ (которые создают выпуклости мышц).

## 3.5. Зоны перекрытия (blending между костями)

На стыках костей — ОБЯЗАТЕЛЬНО перекрытие вокселей. Без перекрытия Surface Nets создаст ПЕРЕТЯЖКУ (тонкое место) на стыке — «шея Буратино».

```rust
fn fill_body(skeleton: &Skeleton, profiles: &BoneProfiles, grid: &mut VoxelGrid) {
    for bone in &skeleton.bones {
        let profile = &profiles[bone.id];
        
        // Заполняем воксели вдоль кости
        // НО: t идёт от -0.15 до 1.15 (вылезает за пределы кости на 15%)
        let t_start = -0.15;  // захватываем ПРЕДЫДУЩУЮ кость
        let t_end = 1.15;     // захватываем СЛЕДУЮЩУЮ кость
        
        for t in frange(t_start, t_end, step) {
            let (rx, rz) = profile.section_at(t.clamp(0.0, 1.0));
            
            // Плавное затухание за пределами кости
            let fade = if t < 0.0 { 1.0 + t / 0.15 }    // 0→1
                  else if t > 1.0 { 1.0 - (t - 1.0) / 0.15 } // 1→0
                  else { 1.0 };
            
            let rx_faded = rx * fade;
            let rz_faded = rz * fade;
            
            // Заполнить сечение
            fill_section(grid, bone, t, rx_faded, rz_faded);
        }
    }
}
```

Перекрытие 15% с каждой стороны создаёт зону, где воксели двух соседних костей СМЕШИВАЮТСЯ. Surface Nets обрабатывает это как один непрерывный объём — и стык исчезает.

# ЧАСТЬ 4: ПОЛНЫЙ РЕЦЕПТ — ОТ БУРАТИНО К ЧЕЛОВЕКУ

## Шаг 1: Исправь пропорции (числа из предыдущего документа)
Длины костей, радиусы — подставь новые. Это уберёт самолёт и ведро.

## Шаг 2: Замени эллипсы на суперэллипсы
Добавь параметр `n` к каждому BodySection. Торс n=2.3, кисть n=3.0, всё остальное n=2.0. Это уберёт «колбасность».

## Шаг 3: Добавь offset_z
Грудь вперёд (+1.0), затылок назад (-1.0), икры назад (-1.0). Это создаст асимметрию глубины. Тело перестанет быть бревном.

## Шаг 4: Добавь мышечные подушки
Начни с трёх: грудные, дельтовидные, квадрицепсы. Если эффект заметен и красив — добавь остальные. Это создаст АНАТОМИЧЕСКИЕ ПЕРЕГИБЫ вместо гладких труб.

## Шаг 5: Включи перекрытие 15% на стыках костей
Воксели соседних костей должны смешиваться. Это уберёт «шею Буратино» и «паховую щель».

## Шаг 6: (Опционально) Градиентная плотность вместо бинарной
Если Surface Nets поддерживает scalar field — используй. Радикальное улучшение гладкости без увеличения разрешения.

## Шаг 7: Компенсация Laplacian shrinkage
Увеличь все радиусы на 3-5%. Laplacian сожмёт обратно.

# ЧАСТЬ 5: САМОЕ ВАЖНОЕ — КАК ПРОВЕРЯТЬ

После каждого изменения — ЧЕТЫРЕ РАКУРСА:

1. **Спереди.** Ищи: песочные часы (плечи-талия-бёдра). Чередование (толстое-тонкое на каждой конечности). Голова = яйцо, не ведро.

2. **Сбоку.** Ищи: S-кривую (грудь вперёд, поясница назад). Ягодицы выступают. Икры выступают. Затылок выступает. Если тело — доска, добавь offset_z.

3. **Сверху.** Ищи: грудная клетка = скруглённый прямоугольник (n=2.3), не блин (rz слишком мал) и не круг (n=2.0 на торсе). Голова = овал, длинная ось вперёд-назад (затылок!).

4. **Три четверти (45°).** Ищи: ПЛАВНОСТЬ перетеканий. Если видишь стыки между костями — увеличь перекрытие. Если видишь ступеньки — увеличь разрешение или Laplacian итерации. Если видишь «мыло» — уменьши Laplacian blend.

И СРАВНИВАЙ с референсами Жени. Не числа — СИЛУЭТ. Совпадает ли контур? Есть ли те же выпуклости и впадины? Если нет — крути параметры пока не совпадёт.
