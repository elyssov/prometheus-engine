## 1. PROMETHEUS SCRIPT — декларативный язык для ИИ-кодеров

Вместо Rust-кода — YAML/TOML описание сцен, персонажей, поведения. ИИ генерирует текст, движок интерпретирует.

```yaml
entity:
  name: "ORPP Soldier"
  skeleton: human
  body: athletic
  clothing: [orpp_coat, tactical_pants, combat_boots]
  weapon: ak12
  position: [128, 92, 128]
  behavior:
    - patrol: [point_a, point_b, point_c]
    - on_detect: aim_and_fire
    - on_hit: take_cover
```

**Почему:** ИИ не должен писать `Quat::from_axis_angle(Vec3::X, 0.087)`. ИИ должен писать `knee_bend: 5deg`. Декларативный барьер между "что" и "как".

## 2. HOT RELOAD — изменения без перезапуска

Изменил .prom файл (модель, текстуру, поведение) → движок подхватывает мгновенно. Без перекомпиляции. Файловый вотчер + горячая замена ассетов.

**Почему:** итерации. 20 блинов за день — потому что каждый rebuild = 3 секунды. С hot reload — 0 секунд.

---

## 3. VISUAL SKELETON EDITOR — UI для "мясных мешков"

Окно: 3D-вьюпорт с воксельным персонажем. Суставы — цветные сферы, которые можно тягать мышкой. Drag сустав → IK пересчитывает → модель обновляется в реальном времени.

Панели: список костей, слайдеры углов, кнопки поз (T-pose, стрельба, присед). Export в .prom.

**Почему:** дизайнер анимаций НЕ должен писать код. Он должен тягать мышкой.

---

## 4. PROCEDURAL MATERIAL SYSTEM — материалы из математики

Вместо текстурных файлов — процедурные шейдеры для поверхностей. Камуфляж = Perlin noise с 3 цветами. Дерево = grain pattern из Simplex noise. Металл = reflection + Worley noise царапин.

```rust
material("woodland_camo") {
    base: noise(perlin, scale=0.3, colors=[olive, brown, black])
    roughness: 0.7
    wear: noise(worley, scale=0.1, intensity=0.3)
}
```

**Почему:** нулевой размер текстур. Бесконечная вариативность. kkrieger-подход.

---

## 5. DESTRUCTION GRAPH — граф связности для реалистичного разрушения

При попадании пули: не просто удаляем сферу вокселей, а:
1. Удаляем в зоне импакта
2. Flood fill от центра масс объекта
3. Отделившиеся куски → новые физические объекты
4. Мелкие куски → частицы
5. Большие куски → падают с физикой, могут разрушить другие объекты

**Почему:** ваза падает со стола и РАЗБИВАЕТСЯ НА КУСКИ. Не исчезает. Не становится "повреждённой текстурой". Разбивается.

---

## 6. SOUND FROM MATERIAL — звук из физики материала

Каждый материал имеет акустические свойства. При столкновении: звук генерируется процедурно из массы, скорости, материалов обоих объектов.

Стекло × пол = звон. Дерево × дерево = стук. Металл × камень = лязг. Не записанные сэмплы — синтезированные звуки.

**Почему:** бесконечная вариативность звуков. Каждый удар уникален.

---

## 7. BEHAVIOR TREES С ВИЗУАЛЬНЫМ РЕДАКТОРОМ

Нодовый редактор поведения: Sequence, Selector, Condition, Action. Drag & drop в UI. Пёс: [Sleep → OnNoise → WakeUp → Patrol → ReturnToSleep]. Кот: [Wander → SeeObject → Approach → Push → Watch Fall → Purr].

**Почему:** гейм-дизайнер рисует поведение мышкой. Не пишет код.

---

## 8. MULTI-ENTITY SCENES С ВЗАИМОДЕЙСТВИЕМ

Несколько Entity в одной сцене. Каждый знает о других. Кот видит пса → крадётся. Пёс слышит шум → просыпается. Солдат видит врага → стреляет.

Perception system: зрение (raycast), слух (расстояние + громкость), обоняние (для животных).

**Почему:** одинокий солдат в комнате — демо. Два персонажа, которые реагируют друг на друга — игра.

---

## 9. VOXEL TERRAIN — ландшафт из вокселей

Не только комнаты — открытые пространства. Terrain из noise: холмы, долины, скалы. Деревья из L-systems. Вода из simplified fluid. Разрушаемый (взрыв оставляет кратер).

Для ОРПП:Апокалипсис: город Скобелевск из процедурных зданий + улиц + terrain.

**Почему:** масштаб. Комната → здание → квартал → город.

---

## 10. NETWORK MULTIPLAYER FOUNDATION

Voxel diff protocol: вместо передачи всего grid — только изменённые воксели. Delta compression. При разрушении стены: передаём координаты удалённых вокселей, не весь grid.

Authoritative server: физика на сервере, рендер на клиенте.

**Почему:** PURRGE "Два кота" (кооп). ОРПП:Апокалипсис — тактический мультиплеер.

---

## 11. REPLAY SYSTEM — запись и воспроизведение

Записываем не видео — записываем inputs + entity states. Воспроизводим: камера свободная, можно смотреть с любого ракурса. Slow-mo, pause, rewind.

Replay файл = последовательность (timestamp, entity_id, action). Мегабайты, не гигабайты.

**Почему:** PURRGE — "поделиться чеком". Но что если поделиться ВИДЕО разрушения с любого ракурса? Вирусный потенциал ×100.

---

## 12. PHOTO-TO-VOXEL PIPELINE — конвертер из фотографий

Уже описан в архитектуре. Фото фронтальное + боковое → маркеры суставов → скелет → профили тела → воксельный ассет.

Следующий уровень: ИИ-модель (MediaPipe/OpenPose) автоматически размещает маркеры. Один клик = воксельный персонаж из фото.

**Почему:** контент-пайплайн. Дизайнер не моделирует — фотографирует.

---

## 13. ANIMATION MARKETPLACE — обмен анимациями

Стандартный формат анимаций (.panim — Prometheus Animation). Набор keyframes для стандартного скелета. Любая анимация работает на любом персонаже (retargeting через пропорции).

Marketplace: скачал "tactical_reload.panim" → работает на солдате, на полицейском, на гражданском. Разные тела — одна анимация.

**Почему:** создание анимаций — самое трудоёмкое. Переиспользование — экономия месяцев.

---

## 14. EMBEDDED AI DIRECTOR — ИИ-режиссёр в движке

Claude/GPT API встроен в движок. Описываешь сцену текстом → ИИ генерирует Prometheus Script → движок исполняет.

"Солдат входит в комнату, осматривается, замечает врага за столом, приседает, стреляет три раза, враг падает, солдат проверяет тело."

ИИ конвертирует в последовательность entity.walk(), entity.aim_at(), entity.fire().

**Почему:** кат-сцены создаются за минуты, не за недели. Режиссёр описывает словами — ИИ ставит сцену.

---

## 15. CROSS-COMPILE EVERYWHERE — один код, все платформы

Rust + wgpu уже даёт: Windows, macOS, Linux, Web (WASM). Добавить:
- Android (через NDK)
- iOS (через Metal)
- Nintendo Switch (через NVN/Vulkan)
- Steam Deck (нативный Linux)

Один `cargo build --target=...` → бинарник для платформы.

**Почему:** PURRGE на Switch. ОРПП:Апокалипсис на PC. Демо в браузере. Один движок.
