# 1. ЦИКЛ ХОДЬБЫ — УГЛЫ СУСТАВОВ НОГ

## 1.1. Восемь фаз gait cycle

Один полный цикл = 0-100%. Правая нога — опорная от 0% до 60% (stance phase), свободная от 60% до 100% (swing phase). Левая — в противофазе, сдвинута на 50%.

```
0%    Initial Contact (IC)     — пятка касается земли
0-12% Loading Response (LR)    — вес переносится на ногу
12-31% Mid Stance (MSt)        — нога под центром тяжести
31-50% Terminal Stance (TSt)   — пятка отрывается, перекат на носок
50-62% Pre-Swing (PSw)         — носок отрывается
62-75% Initial Swing (ISw)     — нога выносится вперёд
75-87% Mid Swing (MSw)         — нога проходит под телом
87-100% Terminal Swing (TSw)   — нога выпрямляется для следующего IC
```

## 1.2. Таблица углов (градусы)

Конвенция: сгибание (flexion) = положительные значения, разгибание (extension) = отрицательные. Ноль = анатомическая позиция (стоя прямо).

### Бедро (Hip)

| Фаза | % цикла | Угол (°) | Описание |
|------|---------|----------|----------|
| IC | 0% | +30 | Бедро согнуто, нога впереди |
| LR | 6% | +25 | Начало переноса веса |
| MSt | 20% | +10 | Нога проходит вертикаль |
| TSt | 40% | -10 | Бедро разгибается, нога позади |
| PSw | 55% | -10 | Максимальное разгибание |
| ISw | 65% | +15 | Бедро начинает сгибаться |
| MSw | 80% | +25 | Бедро сгибается, нога выносится |
| TSw | 95% | +30 | Возврат к IC |

**Диапазон: от -10° до +30°. Амплитуда ~40°.**

### Колено (Knee)

| Фаза | % цикла | Угол (°) | Описание |
|------|---------|----------|----------|
| IC | 0% | +5 | Почти прямое, лёгкое сгибание |
| LR | 6% | +15 | Амортизация удара (важно!) |
| MSt | 20% | +5 | Выпрямление под весом |
| TSt | 40% | +5 | Прямое |
| PSw | 55% | +35 | Начало сгибания для отрыва |
| ISw | 65% | +60 | Максимальное сгибание (нога укорачивается для переноса) |
| MSw | 80% | +30 | Разгибание |
| TSw | 95% | +5 | Почти прямое, готово к IC |

**Диапазон: от +5° до +60°. Амплитуда ~55°.**

**КРИТИЧНО для тебя, Лара:** колено НИКОГДА не уходит в 0° или отрицательные значения при ходьбе. Минимум +5°. Если у тебя колени «переразгибаются» — clamp на 5° минимум.

### Голеностоп (Ankle)

| Фаза | % цикла | Угол (°) | Описание |
|------|---------|----------|----------|
| IC | 0% | 0 | Нейтральная позиция |
| LR | 6% | -5 | Стопа опускается (plantarflexion) |
| MSt | 20% | +5 | Голень наклоняется вперёд (dorsiflexion) |
| TSt | 40% | +10 | Максимальная dorsiflexion |
| PSw | 55% | -20 | Отталкивание! Максимальная plantarflexion |
| ISw | 65% | 0 | Возврат к нейтральной |
| MSw | 80% | 0 | Нейтральная (ноге не за что цепляться) |
| TSw | 95% | 0 | Нейтральная, готово к IC |

**Диапазон: от -20° до +10°. Амплитуда ~30°.**

### Таз (Pelvis)

| Параметр | Амплитуда | Фаза относительно шага |
|----------|-----------|------------------------|
| Tilt (наклон вперёд-назад) | ±4° | Максимальный наклон вперёд при toe-off |
| Obliquity (боковой наклон) | ±5° | Опускается на стороне свободной ноги |
| Rotation (вращение) | ±8° | Вращается ВПЕРЁД на стороне свободной ноги |

Все три — синусоиды с разными фазовыми сдвигами. Это критически важно для натуральности: таз НЕ СТОИТ НА МЕСТЕ. Он качается по всем трём осям.

# 2. ЦИКЛ ХОДЬБЫ — ВЕРХНЯЯ ЧАСТЬ ТЕЛА

## 2.1. Руки

Руки качаются в ПРОТИВОФАЗЕ к ногам. Правая нога вперёд → левая рука вперёд.

### Плечо (Shoulder)

| Фаза ноги | Угол плеча ПРОТИВОПОЛОЖНОЙ руки |
|-----------|-------------------------------|
| IC (0%) | +25° (рука впереди) |
| MSt (20%) | +10° |
| TSt (40%) | -15° (рука позади) |
| PSw (55%) | -20° (максимум назад) |
| MSw (80%) | +15° |
| TSw (95%) | +25° |

**Диапазон: от -20° до +25°. Амплитуда ~45°.**

### Локоть (Elbow)

Локоть НЕ свободно болтается. Он СЛЕГКА сгибается когда рука впереди и почти выпрямляется когда позади.

| Позиция руки | Угол локтя |
|-------------|-----------|
| Впереди максимум | +30° |
| Нейтраль (вдоль тела) | +10° |
| Позади максимум | +5° |

**Формула: elbow_flex = 10 + 20 × max(0, sin(arm_phase))**

Только положительные значения! Локоть НЕ разгибается в отрицательные — это не колено, там замок.

### Важная деталь
Руки качаются НЕ строго вперёд-назад. Есть лёгкое приведение к центру тела (adduction) ≈ 5-10°. Руки слегка пересекают среднюю линию при махе вперёд. Это создаёт натуральность.

## 2.2. Торс

Торс вращается в ПРОТИВОФАЗЕ к тазу. Таз повернулся вправо → плечи повернулись влево. Амплитуда вращения плеч ≈ ±6° (меньше чем таз).

Суммарное «скручивание» (twist) между тазом и плечами = ±14° (8° таза + 6° плеч в противоположном направлении).

## 2.3. Голова

Голова СТАБИЛИЗИРОВАНА. Компенсирует вращение торса. При нормальной ходьбе голова почти не вращается — мозг стабилизирует взгляд. Лёгкое вертикальное покачивание синхронно с шагом (±1-2 см амплитуда, два пика за цикл — по одному на каждый шаг).

**Для процедурной анимации:** `head_rotation = -torso_rotation × 0.8`. Не полная компенсация — 80%. Остаток создаёт лёгкое покачивание головы.

# 3. КИНЕМАТИКА СТОПЫ

## 3.1. Перекат (Rocker Mechanism)

```
   IC (0%)          LR (6%)         MSt (20%)       TSt (40%)       PSw (55%)
     ▼                ▼               ▼               ▼               ▼
   ┌───┐            ┌───┐           ┌───┐           ┌───┐           ┌───┐
   │   │            │   │           │   │           │   │           │   │
   └───┘            └───┘           └───┘           └───┘           └───┘
  /     \          /    \             |   \            \   \              \
 пятка    ──────  вся      ──────   перед   ──────   носок  ──────    отрыв
касается         стопа             часть            только
```

### Угол стопы к земле по фазам

| Фаза | Угол стопы к горизонту | Контакт | Высота пятки | Высота носка |
|------|----------------------|---------|-------------|-------------|
| IC (0%) | +20° (носок вверх) | Пятка | 0 | +3 вокселя |
| LR (6%) | 0° (плоско) | Вся стопа | 0 | 0 |
| MSt (20%) | 0° | Вся стопа | 0 | 0 |
| TSt (35%) | -10° (пятка вверх) | Передняя часть | +2 вокс. | 0 |
| TSt (45%) | -25° (пятка высоко) | Носок | +5 вокс. | 0 |
| PSw (55%) | -30° (максимум) | Кончики пальцев | +7 вокс. | 0 |
| ISw (62%) | 0° (в воздухе) | Нет контакта | +3 вокс. | +3 вокс. |
| MSw (80%) | 0° | Нет контакта | +5 вокс. | +5 вокс. |
| TSw (95%) | +15° | Нет контакта | +2 вокс. | +4 вокс. |

## 3.2. Реализация для 128³ модели

Стопа — 2-3 вокселя в высоту, 6-8 в длину. Разбиваем на два сегмента: пяточный (задний) и носочный (передний). Между ними — сустав. Угол между сегментами = dorsiflexion/plantarflexion.

При IC: пяточный сегмент на земле, носочный поднят на ~20°.
При midstance: оба на земле (0°).
При toe-off: пяточный поднят, носочный на земле, угол -30°.

# 4. ЦИКЛ БЕГА

## 4.1. Ключевые отличия от ходьбы

| Параметр | Ходьба | Бег |
|----------|--------|-----|
| Stance phase | 60% цикла | 40% цикла |
| Swing phase | 40% цикла | 60% цикла |
| Фаза полёта (float) | НЕТ | ЕСТЬ (~15% цикла) |
| Наклон корпуса | 0-2° вперёд | 5-15° вперёд |
| Амплитуда бедра | 40° | 65° |
| Амплитуда колена | 55° | 100°+ |
| Вертикальное колебание ЦТ | 3-5 см | 8-12 см |
| Частота | 1.5-2.0 Гц | 2.5-3.5 Гц |

## 4.2. Углы суставов при беге

### Бедро

| Фаза | % цикла | Угол (°) |
|------|---------|----------|
| IC | 0% | +40 |
| MSt | 15% | +15 |
| Toe-off | 40% | -20 |
| Max flex (swing) | 75% | +50 |
| TSw | 95% | +40 |

**Диапазон: от -20° до +50°. Амплитуда ~70°.**

### Колено

| Фаза | % цикла | Угол (°) |
|------|---------|----------|
| IC | 0% | +20 |
| MSt | 15% | +40 (больше амортизация!) |
| Toe-off | 40% | +20 |
| Max flex (swing) | 70% | +120 (пятка почти касается ягодиц!) |
| TSw | 95% | +15 |

**Диапазон: от +15° до +120°. Амплитуда ~105°.**

### Руки при беге
Локти согнуты на 90° ПОСТОЯННО (не как при ходьбе). Качание — в плечевом суставе. Амплитуда плеча: ±40° (больше чем при ходьбе). Кисти сжаты в кулаки (если применимо к солдату).

## 4.3. Фаза полёта

Обе ноги в воздухе. Происходит дважды за цикл:
- После toe-off опорной ноги (~42-45% цикла)
- После toe-off другой ноги (~92-95% цикла)

Длительность каждой фазы полёта: ~5-8% цикла.

Во время полёта: центр тяжести движется вниз (параболическая траектория). Обе ноги в воздухе. Тело «летит». Приземление — на переднюю часть стопы (НЕ на пятку как при ходьбе!).

# 5. ТАКТИЧЕСКИЙ ПРИСЕД / СТРЕЛЬБА С КОЛЕНА

## 5.1. Tactical Kneeling (стрельба с колена)

Правое колено на земле, левая нога вперёд (для правши).

### Углы

| Сустав | Левая нога (опорная) | Правая нога (на земле) |
|--------|---------------------|----------------------|
| Бедро | +80° (сильно согнуто) | +10° (почти прямое) |
| Колено | +90° (прямой угол) | +120° (голень назад) |
| Голеностоп | +15° (dorsiflexion) | -40° (подъём стопы на земле) |

### Торс
Наклон вперёд: +10-15°. Лёгкий поворот в сторону цели: ±10°.

### Руки (с оружием)
Правое плечо: +80° flexion, +15° abduction.
Правый локоть: +90° (параллельно земле).
Левая рука поддерживает цевьё: плечо +70°, локоть +100°.

## 5.2. Переход «стоя → присед»

Длительность: ~0.5 секунд (15 кадров при 30fps).

Фазы:
1. Кадры 1-5: правая нога начинает сгибаться, корпус опускается
2. Кадры 5-10: правое колено касается земли, вес переносится
3. Кадры 10-15: стабилизация, оружие поднимается к плечу

Кривая: ease-in-out (быстро в середине, плавно в начале и конце).

## 5.3. Полный присед (squat)

Обе ноги согнуты симметрично.

| Сустав | Угол |
|--------|------|
| Бедро | +110° |
| Колено | +140° |
| Голеностоп | +25° |
| Торс | +15° вперёд |

# 6. КРИВЫЕ И IK

## 6.1. Кривые интерполяции

**НЕ используй линейную интерполяцию для суставов.** Линейная = механический робот. Живые суставы двигаются по S-образным кривым.

### Smoothstep (у тебя уже есть)
```
smoothstep(t) = t * t * (3 - 2 * t)
```
Хорошо для переходов между позами. Но для циклической ходьбы — не идеально.

### Для ходьбы — модифицированные синусоиды

Не чистый sin(t), а «сплющенный» синус, который задерживается дольше в крайних точках:

```
// "Пружинистый" синус — задерживается в крайних положениях
spring_sin(t) = sin(t)^0.8 * sign(sin(t))

// Или через clamp для плоских участков (стопа на земле):
clamped_sin(t) = clamp(sin(t) * 1.3, -1.0, 1.0)
```

### Для каждого сустава — своя кривая

| Сустав | Рекомендуемая кривая | Почему |
|--------|---------------------|--------|
| Бедро | sin(phase) | Плавное качание, почти симметричное |
| Колено | abs(sin(phase))² + offset | Колено ТОЛЬКО сгибается (всегда ≥5°), пиковое сгибание резкое |
| Голеностоп | Кусочная: flat → sin → flat | Долго на нуле (стопа на земле), резкий plantarflex при отталкивании |
| Плечо | sin(phase + π) | Противофаза к бедру |
| Локоть | 10 + 20 * max(0, sin(arm_phase)) | Только положительные значения |

### Конкретная формула для колена при ходьбе

```
fn knee_angle(phase: f32) -> f32 {
    // phase: 0.0 - 1.0 (полный gait cycle)
    let stance_flex = if phase < 0.12 {
        // Loading response: амортизационное сгибание
        15.0 * sin(phase / 0.12 * PI)
    } else if phase < 0.5 {
        // Mid-to-terminal stance: почти прямое
        5.0
    } else {
        0.0
    };
    
    let swing_flex = if phase >= 0.5 && phase < 0.85 {
        // Swing: большое сгибание
        let t = (phase - 0.5) / 0.35;
        60.0 * sin(t * PI)
    } else if phase >= 0.85 {
        // Terminal swing: разгибание
        let t = (phase - 0.85) / 0.15;
        60.0 * (1.0 - t) * sin(PI)
    } else {
        0.0
    };
    
    5.0 + stance_flex + swing_flex  // Минимум 5° всегда!
}
```

## 6.2. Inverse Kinematics (IK) для ног

IK критичен когда стопа должна стоять на поверхности определённой высоты (ступенька, наклон).

### Two-Bone IK (бедро-колено-стопа)

```
// Входные данные:
//   hip_pos: позиция тазобедренного сустава (известна)
//   target_pos: позиция стопы (определяется поверхностью)
//   thigh_len: длина бедра
//   shin_len: длина голени

fn solve_two_bone_ik(hip_pos: Vec3, target_pos: Vec3, 
                      thigh_len: f32, shin_len: f32) -> (f32, f32) {
    let dist = (target_pos - hip_pos).length();
    
    // Ограничение: стопа не может быть дальше чем бедро + голень
    let dist = dist.min(thigh_len + shin_len - 0.01);
    
    // Ограничение: стопа не может быть ближе чем |бедро - голень|
    let dist = dist.max((thigh_len - shin_len).abs() + 0.01);
    
    // Угол бедра (закон косинусов)
    let hip_angle = acos(
        (thigh_len * thigh_len + dist * dist - shin_len * shin_len)
        / (2.0 * thigh_len * dist)
    );
    
    // Угол колена (закон косинусов)
    let knee_angle = PI - acos(
        (thigh_len * thigh_len + shin_len * shin_len - dist * dist)
        / (2.0 * thigh_len * shin_len)
    );
    
    // hip_angle — угол между вектором hip→target и бедром
    // knee_angle — угол сгибания колена (0 = полностью прямое)
    
    // Добавляем угол от hip_pos к target_pos в мировых координатах
    let dir = (target_pos - hip_pos).normalize();
    let world_angle = atan2(dir.z, -dir.y); // зависит от системы координат
    
    (world_angle + hip_angle, knee_angle)
}
```

### Pole Vector (куда смотрит колено)

Колено должно смотреть ВПЕРЁД. Pole vector = точка перед персонажем. При повороте — pole vector поворачивается вместе.

```
let pole_target = hip_pos + character_forward * 10.0;
```

## 6.3. Foot Planting (привязка стопы к земле)

Во время stance phase стопа НЕ ДОЛЖНА скользить по земле. Алгоритм:

```
1. При IC (пятка касается): запомнить мировую позицию стопы (foot_plant_pos)
2. Пока stance phase: стопа = foot_plant_pos (не двигается!)
3. Тело движется ВПЕРЁД → IK пересчитывает углы бедра и колена
4. При toe-off: стопа освобождается, начинает swing-движение
```

Это предотвращает «скольжение» стопы — главный артефакт плохой процедурной ходьбы.

# 7. КИНЕМАТИКА КОТА

## 7.1. Кошачья ходьба (diagonal/trotting gait)

Кот ходит «диагональной рысью»: правая передняя + левая задняя одновременно, потом наоборот.

```
Фазовые сдвиги лап (0.0 - 1.0):
  Правая передняя (RF): 0.0
  Левая задняя (LH):    0.0 (синхронно с RF)
  Левая передняя (LF):  0.5
  Правая задняя (RH):   0.5 (синхронно с LF)
```

### Углы суставов передней лапы

| Фаза | Плечо | Локоть | Запястье |
|------|-------|--------|----------|
| Contact (0%) | +20° | +30° | 0° |
| Support (25%) | 0° | +15° | 0° |
| Lift-off (50%) | -15° | +20° | -10° |
| Swing (75%) | +30° | +45° | +5° |

### Углы суставов задней лапы

| Фаза | Бедро | Колено | Скакательный |
|------|-------|--------|-------------|
| Contact (0%) | +25° | +40° | +15° |
| Support (25%) | +10° | +30° | +5° |
| Lift-off (50%) | -10° | +35° | -15° |
| Swing (75%) | +35° | +60° | +20° |

### Позвоночник

Кошачий позвоночник ГИБКИЙ — волнообразное движение при ходьбе. Сегменты позвоночника (spine1, spine2, spine3) волнообразно отклоняются:

```
spine_lateral[i] = 3.0 * sin(phase * 2π + i * 0.3)  // градусы
```

Каждый следующий сегмент — с запаздыванием 0.3 радиана. Создаёт «змейку» — волна проходит от плеч к тазу.

## 7.2. Кошачий бег (rotary gallop)

При галопе кот использует позвоночник как ПРУЖИНУ:

```
Фазы:
1. Задние лапы отталкиваются → позвоночник РАЗГИБАЕТСЯ (вытянут)
2. Фаза полёта 1 (вытянутый)
3. Передние лапы приземляются → позвоночник СГИБАЕТСЯ (сжат)
4. Фаза полёта 2 (сжатый)
```

Амплитуда сгибания позвоночника при галопе: ±20° (гораздо больше чем при ходьбе!). Это УДВАИВАЕТ длину шага — кошачий секрет скорости.

Лапы при галопе работают парами НЕ диагонально:
```
Задние обе → полёт → передние обе → полёт
```

## 7.3. Кошачий прыжок

### Подготовка (preparation, 0.3 сек)
- Задние ноги: бедро +90°, колено +130° (глубокий присед)
- Передние ноги: слегка согнуты
- Позвоночник: сжат (spine flex +25°)
- Попа вибрирует (±2° быстрый тремор на задних лапах, 10 Гц) — ЭТО ВАЖНО ДЛЯ КАВАЙ

### Отталкивание (launch, 0.1 сек)
- Задние ноги: резкое разгибание (колено: 130° → 10° за 0.1 сек)
- Позвоночник: разгибается (spine flex: +25° → -10°)
- Передние лапы: отрываются первыми
- Кривая: ease-out (быстрое ускорение, потом замедление)

### Полёт (flight, переменная)
- Все лапы подогнуты к телу
- Хвост: баланс (вращение в противоположную сторону от вращения тела)
- Позвоночник: вытянут

### Приземление (landing, 0.15 сек)
- Передние лапы ПЕРВЫМИ (кошачья суперсила!)
- Амортизация: плечо и локоть сгибаются, поглощая удар
- Задние лапы: через 0.05 сек после передних
- Позвоночник: сжимается (амортизация)

## 7.4. Крадущийся кот (stealth walk)

- Высота тела: на 40% ниже обычной (сильнее согнуты все суставы)
- Скорость: 30% от нормальной ходьбы
- Лапы: поднимаются ОЧЕНЬ высоко (колено до +80°) и ставятся ОЧЕНЬ аккуратно
- Позвоночник: низкий, горизонтальный
- Хвост: низко, медленные горизонтальные движения
- Уши: вперёд (pointing forward)
- Глаза: расширены (для кавай-модели — БОЛЬШИЕ ЗРАЧКИ)
- Попа виляет перед финальным прыжком (та самая вибрация)

```
fn stealth_walk(phase: f32) -> Pose {
    let mut pose = normal_walk(phase);
    
    // Опустить тело
    pose.spine_height *= 0.6;
    
    // Увеличить подъём лап

    pose.rear_knee_swing_max = 70.0;   // вместо 60°

    pose.speed *= 0.3;
    
    // Хвост низко
    pose.tail_base_angle = -10.0;  // вместо +5°

}
```

## 7.5. Хвост (физическая цепочка)

Хвост — 4 кости (tail1-tail4). Каждая следующая — длиннее и легче. Симулируется как ВЕРЛЕТОВСКИЙ ВЕРЁВОЧНЫЙ МАЯТНИК:

```
fn simulate_tail(bones: &mut [TailBone; 4], dt: f32) {
    // Гравитация
    for bone in bones.iter_mut() {
        bone.velocity.y -= 9.8 * dt;
    }
    
    // Демпфирование (трение воздуха)
    for bone in bones.iter_mut() {
        bone.velocity *= 0.95;
    }
    
    // Ограничения (каждая кость привязана к предыдущей)
    for i in 1..4 {
        let dir = (bones[i].pos - bones[i-1].pos).normalize();
        let target = bones[i-1].pos + dir * bone_length;
        bones[i].pos = target;
    }
    
    // Ограничение углов (хвост не может изгибаться на 180°)
    for i in 1..4 {
        let angle = angle_between(bones[i-1], bones[i]);
        if angle > MAX_TAIL_BEND {
            // clamp
        }
    }
}
```

**Настроение кота через хвост:**
- Довольный: хвост вверх (+30° base), кончик слегка загнут, медленные мягкие качания
- Охотится: хвост горизонтально (0°), кончик подёргивается (±5° быстрый тремор)
- Испуган: хвост распушён (увеличить радиус вокселей ×1.5), дугой вверх
- Раздражён: хвост хлещет из стороны в сторону (±30°, быстро)
- Расслаблен: хвост свисает (-20°), обвивает лапы когда сидит

## 8.3. Пружинистость (bounce)

Добавить к вертикальному положению тела лёгкий bounce-эффект. Не постоянный — при приземлении стопы:

```rust
fn bounce(phase: f32) -> f32 {
    // Два «удара» за цикл (каждый шаг)
    let impact = abs(sin(phase * 2.0 * PI));
    
    // Быстрое сжатие, медленное восстановление
    let bounce = impact.powf(3.0) * -2.0; // пикселей вниз
    
    // Добавляем к высоте центра тяжести
    bounce
}
```

`powf(3.0)` делает сжатие РЕЗКИМ (быстро вниз, медленно вверх). Это создаёт ту самую «пружинистость» которой не хватает.

```rust
// Каждый кадр анимации (15 FPS)
fn update_animation(character: &mut Character, dt: f32) {
    // 1. Обновить фазу
    character.walk_phase += dt * character.walk_speed;
    character.walk_phase %= 1.0;
    
    let phase = character.walk_phase;
    
    // 2. Рассчитать позу из формул
    let target_pose = match character.state {
        State::Walk => walk_pose(phase, character.speed),
        State::Run => run_pose(phase, character.speed),
        State::Crouch => crouch_pose(),
        State::Idle => idle_pose(character.idle_timer),
    };
    
    // 3. IK для стоп (привязка к земле)
    let ground_height_left = raycast_down(character.left_foot_pos);
    let ground_height_right = raycast_down(character.right_foot_pos);
    
    if character.left_foot_on_ground {
        let (hip_a, knee_a) = solve_two_bone_ik(
            character.left_hip_pos,
            character.left_foot_planted_pos,
            THIGH_LEN, SHIN_LEN
        );
        target_pose.l_hip = hip_a;
        target_pose.l_knee = knee_a;
    }
    
    // 4. Интерполяция к целевой позе (smoothstep)
    character.current_pose = lerp_pose(
        character.current_pose,
        target_pose,
        smoothstep(dt * BLEND_SPEED)
    );
    
    // 5. Применить позу к скелету
    apply_pose_to_skeleton(character);
    
    // 6. Пересобрать воксельную модель из позиций суставов
    rebuild_voxel_mesh(character);
}

// Пересборка вокселей из скелета
fn rebuild_voxel_mesh(character: &mut Character) {
    character.voxels.clear();
    
    for bone in &character.skeleton.bones {
        let start = bone.world_pos;
        let end = bone.child_world_pos;
        let radius = bone.radius; // в вокселях
        
        // Рисуем цепочку сфер от start до end
        let steps = (start.distance(end) / (radius * 0.5)) as usize;
        for i in 0..=steps {
            let t = i as f32 / steps as f32;
            let center = lerp(start, end, t);
            fill_sphere(&mut character.voxels, center, radius);
        }
    }
    
    // Обновить octree персонажа
    character.octree.rebuild(&character.voxels);
}
```
