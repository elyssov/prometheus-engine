## ФИЛОСОФИЯ

Движок — не набор хаков для демо. Движок — система, которая СЧИТАЕТ ЗА ТЕБЯ.

Ты говоришь: "человек ростом 180, худощавый, в бронеплаще, с АК, повернись к столу и стреляй".
Движок: строит скелет → натягивает тело → одевает → вешает обвес → вставляет оружие → поворачивает через IK → рассчитывает позицию дула → стреляет.

Ты НЕ считаешь координаты. Никогда.

## 1. СЛОИСТАЯ СИСТЕМА (LAYER STACK)

Каждый персонаж — стек слоёв, каждый слой — отдельный ассет:

```
┌─────────────────────────┐
│ ОРУЖИЕ                  │  Привязано к точке на кости (hand_r, back, hip)
├─────────────────────────┤
│ ОБВЕС                   │  Пояс, подсумки, кобура, рация, Форсаж
├─────────────────────────┤
│ БРОНЯ                   │  Бронеплащ, шлем, наколенники, перчатки
├─────────────────────────┤
│ ОДЕЖДА                  │  Штаны, рубашка, ботинки
├─────────────────────────┤
│ ТЕЛО                    │  Воксельная "кожа" — анатомия, мускулатура
├─────────────────────────┤
│ СКЕЛЕТ                  │  Кости + суставы + кинематические ограничения
└─────────────────────────┘
```

### Как это работает

1. **Скелет** — дерево костей с ограничениями (колено: 5°-130°, только одна ось)
2. **Тело** — для каждой кости задаётся "профиль": набор сечений (эллипсы) вдоль кости. Бедро — толстое вверху, тоньше внизу. Торс — широкий в плечах, узкий в талии, плоский спереди-назад
3. **Одежда** — то же самое но с offset от тела. Штаны = профиль ноги + 2 вокселя наружу. Свой материал/цвет
4. **Броня** — ещё один offset. Бронеплащ = торс + 3 вокселя наружу, свой материал (тяжёлый, матовый)
5. **Обвес** — точечные привязки к костям. Подсумок = box привязан к hip_r, смещение (5,0,0)
6. **Оружие** — привязка к hand_r (или back, или hip_l). Имеет точки: muzzle, scope, grip, magazine

### Комбинаторика

Один скелет "Человек" → 
  Тела: "Худой", "Атлет", "Крупный", "Женское"
  Одежда: "Камуфляж", "Гражданская", "Форма ОРПП"
  Броня: "Бронеплащ", "Лёгкий жилет", "Экзоскелет Витязь-3"
  Обвес: "Штурмовой набор", "Разведчик", "Сапёр"
  Оружие: "АК-12", "Вихрь", "ШПК", "Вепрь-МА-12"

Все комбинации работают автоматически. Дизайнер не рисует каждую. Движок собирает.

## 2. СКЕЛЕТ (ПОДРОБНО)

### Кость (Bone)

```rust
struct Bone {
    id: BoneId,
    name: String,
    parent: Option<BoneId>,
    
    // Анатомические данные (не меняются)
    rest_length: f32,              // длина кости в покое
    rest_rotation: Quat,           // ориентация в покое (T-поза)
    
    // Ограничения (joint constraints)
    constraint: JointConstraint,
    
    // Текущее состояние (меняется каждый кадр)
    local_rotation: Quat,          // вращение относительно родителя
    world_position: Vec3,          // мировая позиция (вычисляется)
    world_rotation: Quat,          // мировая ориентация (вычисляется)
}

enum JointConstraint {
    Free,                           // полная свобода (шея, запястье)
    Hinge {                         // одна ось (колено, локоть)
        axis: Vec3,                 // ось вращения
        min_angle: f32,             // минимальный угол (радианы)
        max_angle: f32,             // максимальный
    },
    BallSocket {                    // шаровой (плечо, бедро)
        cone_angle: f32,            // максимальный угол отклонения
        twist_min: f32,             // скручивание мин
        twist_max: f32,             // скручивание макс
    },
}
```

### Forward Kinematics

```
1. Начать с root (pelvis)
2. Для каждой кости по дереву (BFS):
   world_rotation = parent.world_rotation × local_rotation
   world_position = parent.world_position + parent.world_rotation × (0, rest_length, 0)
3. Каждая кость знает свою мировую позицию и ориентацию
```

### Стандартные скелеты

**Человек**: 22 кости (pelvis, spine, chest, neck, head, 2×shoulder, 2×upper_arm, 2×forearm, 2×hand, 2×hip, 2×thigh, 2×shin, 2×foot, 2×toe)

**Кот**: 26 костей (аналогично + хвост 4 сегмента + уши 2)

## 3. ПРОФИЛЬНОЕ ТЕЛО (BODY PROFILES)

### Идея

Каждая кость имеет "профиль" — как выглядит сечение тела вокруг этой кости. Вместо "сфера радиуса R" — эллипс, который меняется вдоль длины кости.

```rust
struct BodyProfile {
    bone_id: BoneId,
    sections: Vec<BodySection>,  // от начала к концу кости
    material: MaterialId,
}

struct BodySection {
    t: f32,              // 0.0 = начало кости, 1.0 = конец
    radius_x: f32,       // радиус по оси X (ширина)
    radius_z: f32,       // радиус по оси Z (глубина)
    offset: Vec2,        // смещение центра сечения от оси кости
}
```

### Пример: бедро

```
t=0.0 (бедро у таза):  radius_x=6, radius_z=5  — толстое
t=0.3:                 radius_x=5.5, radius_z=4.5
t=0.7:                 radius_x=4, radius_z=4    — сужается
t=1.0 (у колена):      radius_x=4.5, radius_z=4.5 — чуть шире (колено)
```

### Пример: торс (spine → chest)

```
t=0.0 (талия):    radius_x=8, radius_z=5    — узкая талия, плоский
t=0.5 (рёбра):    radius_x=10, radius_z=6   — шире
t=1.0 (плечи):    radius_x=12, radius_z=5   — самый широкий, всё ещё плоский
```

### Растеризация

Для каждой кости:
1. Интерполировать секции вдоль длины
2. В каждой точке: заполнить эллипс в плоскости, перпендикулярной оси кости
3. Ось кости = мировое направление кости (из forward kinematics)

Это автоматически даёт анатомически правильное тело при ЛЮБОЙ позе.

## 4. ОДЕЖДА И БРОНЯ (CLOTHING LAYERS)

### Принцип

Одежда = тело + offset. Штаны = профиль ног + 1-2 вокселя наружу. Куртка = профиль торса + 2-3 вокселя.

```rust
struct ClothingLayer {
    name: String,
    covers: Vec<BoneId>,           // какие кости покрывает
    offset: f32,                   // сколько вокселей добавлять поверх тела
    material: MaterialId,
    color_pattern: ColorPattern,   // камуфляж, однотонный, полоски
}

enum ColorPattern {
    Solid(Color),
    Camo { base: Color, spots: Color, scale: f32 },
    Striped { colors: Vec<Color>, width: f32 },
    Noise { base: Color, variation: f32 },       // процедурный шум
}
```

### Пример: бронеплащ ОРПП

```
ClothingLayer {
    name: "Бронеплащ",
    covers: [chest, spine, upper_arm_l, upper_arm_r, hip_l, hip_r, thigh_l, thigh_r],
    offset: 3.0,  // толстый
    material: BALLISTIC_FABRIC,
    color_pattern: Solid(Color::new(75, 78, 85)),  // тёмно-серый
}
```

### Растеризация

1. Для каждой покрытой кости: взять профиль тела
2. Увеличить радиусы на offset
3. Заполнить с материалом/цветом одежды
4. Рисовать ПОВЕРХ тела (перезаписывая внешние воксели)

## 5. КОНВЕРТЕР (ФОТО → АССЕТ)

### Workflow

```
1. Пользователь загружает фото (фронтальный вид, руки разведены — T-поза)
2. SDK показывает фото на экране
3. Пользователь размещает маркеры суставов мышкой:
   - Голова (top, chin)
   - Плечи (L, R)
   - Локти (L, R)
   - Кисти (L, R)
   - Таз (L, R)
   - Колени (L, R)
   - Стопы (L, R)
4. SDK автоматически:
   a. Строит скелет (длины костей из расстояний между маркерами)
   b. Сэмплирует цвета с фото вдоль каждой кости
   c. Оценивает радиусы (ширину конечности) из фото
   d. Генерирует BodyProfile для каждой кости
   e. Создаёт воксельный ассет
5. Пользователь может подкорректировать:
   - Радиусы (толщина конечностей)
   - Цвета
   - Добавить боковое фото для Z-глубины
6. Экспорт: .vox файл (стандартный формат) или .prom (наш формат)
```

### Технически

- Фронтальное фото → X и Y координаты суставов
- Боковое фото (опционально) → Z координаты
- Без бокового: Z оценивается по пропорциям (торс: Z ≈ X×0.5)
- Цвет сэмплируется из пикселей фото вдоль оси кости
- Радиус = ширина конечности в пикселях / 2, масштабированная к воксельному пространству

### Вариант с ИИ (будущее)

Если подключить pose estimation модель (MediaPipe, OpenPose):
1. Загрузить фото
2. ИИ автоматически находит суставы
3. Пользователь только подтверждает/корректирует
4. Генерация ассета одной кнопкой

## 6. СИСТЕМА ПРИВЯЗОК (ATTACHMENTS)

### Точки привязки на скелете

Каждая кость может иметь именованные точки привязки:

```rust
// На кости hand_r:
attach_points: [
    ("grip", Vec3(0, 0, 0)),        // центр хвата
    ("trigger", Vec3(0, -1, 2)),     // палец на спуске
]

// На кости hip_r:
attach_points: [
    ("holster", Vec3(5, 0, 0)),      // кобура справа
]

// На кости back (spine):
attach_points: [
    ("sling", Vec3(0, 0, -3)),       // ремень на спине
]
```

### Оружие как ассет

```rust
struct WeaponAsset {
    name: String,
    voxel_model: VoxelModel,         // воксельная модель оружия
    grip_point: Vec3,                // где рука держит (совмещается с hand.grip)
    muzzle_point: Vec3,              // дуло (откуда вылетает пуля)
    scope_point: Vec3,               // прицел (куда смотрит глаз)
    magazine_point: Vec3,            // магазин
    support_hand_point: Vec3,        // где вторая рука держит (цевьё)
    stock_point: Vec3,               // приклад (к плечу)
}
```

### Автоматическая сборка

```
entity.attach("hand_r.grip", weapon_ak);
// Движок автоматически:
// 1. Совмещает weapon.grip_point с hand_r.grip
// 2. Ориентирует оружие по руке
// 3. Подтягивает левую руку к weapon.support_hand_point (через IK)
// 4. Прижимает weapon.stock_point к плечу (через IK)
// 5. Знает где muzzle → пули вылетают оттуда
```

## 7. ФОРМАТ АССЕТОВ

### .prom — Prometheus Asset Format

```
[Header]
magic: "PROM"
version: 1
type: skeleton | body | clothing | weapon | furniture | room

[Skeleton]
bone_count: u16
bones: [Bone × bone_count]

[Body Profile]
bone_profiles: [BoneProfile × bone_count]

[Clothing Layer]
layer_count: u8
layers: [ClothingLayer × layer_count]

[Voxel Model]  (для оружия, мебели)
size: (u16, u16, u16)
voxels: [Voxel × non_empty_count]  (sparse)
```

### Совместимость

Импорт: .vox (MagicaVoxel), .obj (через воксельизацию)
Экспорт: .vox, .prom, .png (рендер)
